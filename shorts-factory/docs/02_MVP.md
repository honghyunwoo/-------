# Shorts Factory - 최소 기능 제품

## 1. MVP 개요

### 1.1 MVP 정의
YouTube Shorts를 자동 생성하기 위한 최소한의 기능을 갖춘 파이프라인.
명언 입력 → 스크립트 생성 → TTS → 영상 합성 → 출력물 생성

### 1.2 MVP 목표
- **기간**: 7일 (Day 3-9)
- **산출물**: 동작하는 파이프라인 + 첫 영상 10개
- **검증**: 실제 YouTube 업로드 및 성과 확인

### 1.3 MVP 범위
```
┌─────────────────────────────────────────────────────────────┐
│                        MVP 범위                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ✅ 포함 (In Scope)                                        │
│  ├── 스크립트 자동 생성 (Claude API)                       │
│  ├── 재검토 루프                                           │
│  ├── TTS 음성 생성 (TYPECAST)                              │
│  ├── 영상 합성 (MoviePy)                                   │
│  ├── 자막 생성 및 번인                                     │
│  ├── 메타데이터 생성                                       │
│  └── CLI 인터페이스                                        │
│                                                             │
│  ❌ 제외 (Out of Scope)                                    │
│  ├── 썸네일 자동 생성                                      │
│  ├── 배치 생성                                             │
│  ├── 자동 업로드                                           │
│  ├── 성과 추적 대시보드                                    │
│  ├── 영어학습 채널 (2차 개발)                              │
│  └── GUI 인터페이스                                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 기능별 상세 명세

#### MVP-01: 명언 로드
```yaml
기능: 명언 로드
입력: quote_id (정수)
출력: Quote 객체
로직:
  1. quotes_library.json 파일 읽기
  2. quote_id로 명언 검색
  3. Quote 객체 반환
예외:
  - 파일 없음: FileNotFoundError
  - ID 없음: QuoteNotFoundError
```

#### MVP-02: 스크립트 생성
```yaml
기능: 스크립트 생성
입력:
  - quote: Quote 객체
  - hook_type: H1-H5
  - cta_type: C1-C5
출력: Script 객체
로직:
  1. 프롬프트 템플릿 로드
  2. 변수 치환 (명언, 저자, 훅타입, CTA타입)
  3. Claude API 호출
  4. 응답 파싱
  5. Script 객체 생성
API:
  - 모델: claude-sonnet-4-20250514
  - 온도: 0.7
  - 최대 토큰: 500
```

#### MVP-03: 재검토 루프
```yaml
기능: 재검토 루프
입력: Script 객체
출력: 개선된 Script 객체
로직:
  1. 평가 프롬프트로 점수 산출
  2. 평균 8점 이상이면 통과
  3. 미달시 개선 프롬프트 실행
  4. 최대 3회 반복
  5. 최종 스크립트 반환
평가 기준:
  - 훅 강도 (1-10)
  - 명언 정확성 (1-10)
  - 해설 명확성 (1-10)
  - CTA 효과 (1-10)
  - 전체 흐름 (1-10)
```

#### MVP-04: TTS 생성
```yaml
기능: TTS 생성
입력:
  - text: 스크립트 텍스트
  - voice_id: 음성 ID (선택)
  - language: ko/en
출력: audio_path (MP3 파일 경로)
로직:
  1. TTS API 선택 (TYPECAST/ElevenLabs)
  2. API 호출
  3. 오디오 파일 저장
  4. 경로 반환
설정:
  - 샘플레이트: 22050Hz
  - 포맷: MP3
  - 속도: 1.0x
```

#### MVP-05: 자막 생성
```yaml
기능: 자막 생성
입력:
  - script: Script 객체
  - audio_duration: 오디오 길이 (초)
출력: SRT 파일 경로
로직:
  1. 스크립트를 문장 단위로 분리
  2. 문장별 예상 시간 계산
  3. SRT 형식으로 변환
  4. 파일 저장
형식:
  - 자막당 최대 15자 (한글 기준)
  - 2줄 이내
```

#### MVP-06: B-roll 선택
```yaml
기능: B-roll 선택
입력:
  - theme: 테마 (예: 역경, 마음, 시간)
  - duration: 필요 길이
출력: video_clips 리스트
로직:
  1. assets/b-roll 폴더 스캔
  2. 테마 태그로 필터링
  3. 필요 길이에 맞게 클립 선택
  4. 클립 경로 리스트 반환
```

#### MVP-07: 영상 합성
```yaml
기능: 영상 합성
입력:
  - audio_path: 오디오 경로
  - srt_path: 자막 경로
  - video_clips: B-roll 클립 리스트
  - bgm_path: BGM 경로
출력: 최종 영상 경로 (MP4)
로직:
  1. B-roll 클립 연결
  2. 9:16 비율로 리사이즈/크롭
  3. 오디오 합성
  4. 자막 오버레이
  5. BGM 믹싱 (볼륨 20%)
  6. 최종 렌더링
설정:
  - 해상도: 1080x1920
  - FPS: 30
  - 코덱: H.264
  - 비트레이트: 5Mbps
```

#### MVP-08: 메타데이터 생성
```yaml
기능: 메타데이터 생성
입력: Script 객체
출력: metadata.json
로직:
  1. 제목 템플릿 적용
  2. 설명 템플릿 적용
  3. 태그 생성
  4. 해시태그 생성
  5. JSON 저장
템플릿:
  제목: "{명언 핵심 키워드} - {저자} #shorts #스토아"
  설명: |
    {명언 전문}
    - {저자}, {출처}

    #스토아철학 #동기부여 #자기계발 #명언
```

## 5. MVP 테스트 계획

### 5.1 단위 테스트
| 모듈 | 테스트 항목 | 통과 기준 |
|-----|-----------|---------|
| quote_loader | 존재하는 ID 로드 | Quote 객체 반환 |
| quote_loader | 없는 ID 로드 | 예외 발생 |
| script_generator | 스크립트 생성 | 5개 섹션 포함 |
| tts_engine | 오디오 생성 | MP3 파일 생성 |
| video_composer | 영상 합성 | MP4 파일 생성 |

### 5.2 통합 테스트
| 시나리오 | 단계 | 통과 기준 |
|---------|-----|---------|
| 전체 파이프라인 | 명언 ID → 최종 영상 | 10분 내 완료 |
| 오류 복구 | API 실패 시 | 재시도 후 성공 |
| 품질 검증 | 생성된 영상 | 30초 ± 5초 |

### 5.3 수동 검증
| 항목 | 검증 방법 | 기준 |
|-----|---------|-----|
| 스크립트 품질 | 직접 읽기 | 자연스러움 |
| TTS 품질 | 직접 듣기 | 발음 정확도 |
| 영상 품질 | 직접 시청 | 자막 가독성 |
| 업로드 테스트 | YouTube 업로드 | 정상 재생 |

## 6. MVP 성공 기준

### 6.1 기술적 성공
| 기준 | 목표 | 측정 방법 |
|-----|-----|---------|
| 파이프라인 동작 | 100% | 수동 실행 |
