## 🔍 식별된 주요 과제 (Phase 1)

### 과제 #1: 제품 품질의 객관적 측정 불가 (우선순위: 최상)

**문제**: 현재 제품의 품질(예: 비디오 비트레이트, 자막 동기화 정확도)을 측정하는 객관적인 기준과 자동화된 도구가 없어, 개선 작업의 성공 여부를 판단하기 어렵습니다. "감"에 의존한 개발은 재현성과 신뢰성을 떨어뜨립니다.

**해결 계획**:
1.  **품질 검증 스크립트 개발**: FFprobe와 같은 도구를 사용하여 생성된 영상의 비트레이트, 해상도, FPS, 오디오 코덱 등을 자동으로 분석하고 리포트하는 스크립트를 개발합니다.
2.  **자막 정확도 측정 로직 구현**: 생성된 자막(.srt) 파일의 타임스탬프 유효성을 검사하고, 음성 길이와 자막 길이를 비교하여 동기화 정확도를 백분율로 계산하는 로직을 구현합니다.

### 과제 #2: 핵심 기능의 불안정성 (우선순위: 높음)

**문제**: `product_analysis.md`에 언급된 바와 같이, 자막 동기화 실패, 엉뚱한 언어의 음성 합성 등 사용자가 즉시 인지할 수 있는 치명적인 오류가 존재합니다.

**해결 계획**:
1.  **E2E 테스트 케이스 작성**: 오류가 발생하는 시나리오를 그대로 재현하는 End-to-End 테스트 코드를 작성합니다.
2.  **테스트 기반 버그 수정 (TDD)**: 작성된 E2E 테스트가 실패하는 것을 먼저 확인한 후, 테스트를 통과하도록 코드를 수정합니다. 이를 통해 정확한 원인 파악과 재발 방지를 보장합니다.
