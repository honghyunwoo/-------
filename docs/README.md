## 🚀 프로젝트 실행 방법

### 1. 필수 프로그램 설치

- **Python**: 3.10 이상
- **FFmpeg**: 영상 처리를 위해 반드시 필요합니다. [공식 사이트](https://ffmpeg.org/download.html)에서 다운로드 후 시스템 PATH에 추가해주세요.
- **Redis**: 백그라운드 작업 및 캐싱을 위해 필요합니다. [공식 사이트](https://redis.io/download/) 또는 Docker를 통해 설치해주세요.

### 2. 프로젝트 설정

```bash
# 1. 프로젝트 클론
git clone https://your-repository-url/owl-studio.git
cd owl-studio

# 2. 가상환경 생성 및 활성화
python -m venv venv
# Windows
venv\Scripts\activate
# macOS/Linux
# source venv/bin/activate

# 3. 필수 라이브러리 설치
pip install -r requirements.txt

# 4. 설정 파일 복사 및 수정
copy config.example.toml config.toml
```

`config.toml` 파일을 열어 `youtube_api_key`, `openai_api_key` 등 필요한 API 키를 입력해주세요.

### 3. 서버 실행

**총 3개의 터미널이 필요합니다.**

**터미널 1: FastAPI API 서버 실행**
```bash
python main.py
```

**터미널 2: Celery 백그라운드 워커 실행**
```bash
# Windows
celery -A app.worker.celery_app worker --loglevel=info -P gevent

# macOS/Linux
# celery -A app.worker.celery_app worker --loglevel=info
```

**터미널 3: Streamlit 웹 UI 실행**
```bash
streamlit run webui/Main.py
```

### 4. 접속

- **웹 UI**: http://localhost:8501
- **API 문서 (Swagger)**: http://localhost:8080/docs

## 🏗️ 시스템 아키텍처

```mermaid
graph TD
    subgraph "사용자 인터페이스"
        A[Streamlit Web UI]
    end

    subgraph "백엔드 API (FastAPI)"
        B[API Endpoints]
        C[Celery Task Queue]
    end

    subgraph "백그라운드 워커 (Celery)"
        D[Video Generation Worker]
    end

    subgraph "외부 서비스 (External APIs)"
        E[LLM (OpenAI, etc.)]
        F[TTS (Azure, etc.)]
        G[Video Sources (Pexels, etc.)]
        H[YouTube API]
    end

    subgraph "데이터 저장소"
        I[PostgreSQL/SQLite]
        J[Redis (Cache & Broker)]
        K[File Storage (Videos, etc.)]
    end

    A -- HTTP Request --> B
    B -- Enqueue Task --> C
    C -- Broker --> J
    J -- Delivers Task --> D
    D -- Calls --> E
    D -- Calls --> F
    D -- Calls --> G
    D -- Calls --> H
    D -- Read/Write --> I
    D -- Read/Write --> K
    B -- Read/Write --> I
    E -- Caching --> J
    F -- Caching --> J
```

**작동 흐름**:
1.  사용자가 **Streamlit UI**에서 영상 생성을 요청합니다.
2.  **FastAPI**는 요청을 받아 유효성을 검사하고, Celery 작업 큐에 작업을 등록합니다.
3.  **Redis**는 작업을 받아 **Celery 워커**에게 전달합니다.
4.  **Celery 워커**는 실제 영상 생성 로직을 실행하며, 이 과정에서 필요한 외부 API(LLM, TTS, Pexels 등)를 호출합니다.
5.  생성된 영상과 데이터는 **파일 스토리지**와 **데이터베이스**에 저장됩니다.
6.  사용자는 UI를 통해 작업 상태를 확인하고 최종 결과물을 다운로드합니다.
